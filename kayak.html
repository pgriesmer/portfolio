Kayak About Page

Software Engineering Internship, Summer 2018

Overview

As an intern at KAYAK this past summer, I developed new versions of several pages whose links appear in the company's site footer. One suchÂ page was the About page.

Old version
New version

With its outdate appearance, the old version of the About page was doing little to galvanize potential hires to apply for positions with the company.

1 // Hero

After being given a design and assets for the page, I started by building a hero component that now appears atop many of the pages in the footer section of the site. I built this part of the page as a component so that the background image and overlay text could vary from one page to the next.

2 // Slider

Perhaps the most involved part of the page was a slider that appears in the "statistics" section on smaller screen sizes.

After initially configuring the slider to be draggable, I rebuilt it so users would have to click an arrow on either side of the slider to make it slide left or right. The idea for the latter design came from Netflix's streaming menu.

Note how, when three cards are in view (as in the video above), the slider moves three cards at a time. However, when only one card is in view (as below), the slider moves only one card at a time.

I configured the slider to calculate the number of whole "cards" in view at any point in time and slide left or right by the same number of cards with each click of the left- or right-facing arrow.

3 // Offices

Towards the bottom of the page is a section containing information about each of the company's offices. I coded these "office cards" as components so that developers could simply add or remove an office from a list of configurable properties when a new office was opened or closed.

I also built the header for the same section (which only appears on larger screen sizes) to rotate through greetings in the different languages associated with the countries in which KAYAK owned an office. If the company ever adds or removes an office from the configurable list of offices, the greeting associated with that particular office will be automatically added to this rotation.

Outcomes

This project taught me a great deal about web components since nearly every section of this page was its own component. I also learned about all the work it takes to (1) properly configure SEO functionality and (2) make a webpage accessible.
